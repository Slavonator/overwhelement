# API

## Функции

### discretize

```rust
pub fn discretize(scene: &Scene, settings: &Settings) -> ElementBuffer
```

Дискретизирует сцену `Scene` настройками `Settings`, и возвращает `ElementBuffer`

| Аргумент   | Описание                                                                                                         |
| ---------- | ---------------------------------------------------------------------------------------------------------------- |
| `Scene`    | Контейнер высшего уровня. Порядок дискретизации массива `Plane` определяется порядком плоскостей в этом массиве. |
| `settings` | Настройки выходного буфера и фонового слоя                                                                       |

| Результат       | Описание                           |
| --------------- | ---------------------------------- |
| `ElementBuffer` | Контейнер результата дискретизации |

## Трейты

### ElementShader

```rust
pub trait ElementShader {
	fn shade(&self, input: &ShaderInput) -> ShaderOutput;
}
```

Трейт, который обязаны реализовать все шейдеры.

| Аргумент      | Описание                                                     |
| ------------- | ------------------------------------------------------------ |
| `ShaderInput` | Структура, содержащая информацию об обрабатываемом фрагменте |

| Результат      | Описание                                       |
| -------------- | ---------------------------------------------- |
| `ShaderOutput` | Структура, содержащая результат работы шейдера |

## Структуры данных

### Element

```rust
#[derive(Copy, Clone, Debug)]
pub struct Element {
	pub object_id: u32,
	pub depth: f32,
	pub layer: u32,
	pub color: [u8; 3],
	pub luminance: f32,
}
```

Атомарные структуры данных, которые образуются в процессе дискретизации сцены.

| Поля        | Описание                                                                                                           |
| ----------- | ------------------------------------------------------------------------------------------------------------------ |
| `object_id` | Идентификатор объекта, который в итоге оказался в этом элементе                                                    |
| `depth`     | Итоговая видимая глубина элемента. Чем это значение меньше, тем объект ближе к наблюдателю                         |
| `layer`     | Итоговый видимый слой элемента, значение которого указывает на индекс итоговой плоскости в списке плоскостей сцены |
| `color`     | Итоговый видимый цвет элемента                                                                                     |
| `luminance` | Итоговая видимая освещённость элемента                                                                             |

#### Реализации трейтов 

- Сгенерированные компилятором реализации `Copy`, `Clone`, `Debug`

### ElementBuffer

```rust
#[derive(Clone)]
pub struct ElementBuffer {
	pub width: u32,
	pub height: u32,
	pub elements: Vec<Element>,
}
```

Контейнер элементов, который возвращается функцией `discretize`

| Поля              | Описание                                                  |
| ----------------- | --------------------------------------------------------- |
| `width`, `height` | Соответственно ширина и высота выходного буфера элементов |
| `elements`        | Массив элементов                                          |

#### Методы

- `pub fn new(width: u32, height: u32, bg_color: [u8; 3], bg_luminance: f32) -> Self` - возвращает новый `ElementBuffer`, с заданной шириной, высотой, цветом и освещённостью элементов
- `pub fn get(&self, x: u32, y: u32) -> Option<Element>` - возвращает значение `Element` из массива `elements`, по индексу `(y * self.width + x)`

#### Реализации трейтов

- Сгенерированная компилятором реализация `Clone`

### ShaderPool

```rust
#[derive(Clone)]
pub struct ShaderPool {
	pub fallback: Rc<dyn ElementShader>,
}
```

Контейнер всех шейдеров, которые используются на сцене при её дискретизации

| Поля       | Описание                                                                       |
| ---------- | ------------------------------------------------------------------------------ |
| `fallback` | Шейдер, который используется для обработки невалидных ссылок в методе `get()`. |

#### Методы

- `pub fn new() -> Self` - возвращает новый `ShaderPool`, который не имеет шейдеров, и fallback которого установлен как пустой шейдер, возвращающий полностью прозрачный цвет без альфа, который не попадает в `ElementBuffer`
- `pub fn add(&mut self, shader: Rc<dyn ElementShader>) -> u32` - добавляет шейдер в конец пула и возвращает его индекс (отсчёт с нуля)
- `pub fn pop(&mut self) -> Option<Rc<dyn ElementShader>>` - удаляет последний добавленный шейдер и возвращает его, или None, если шейдеров нет
- `pub fn len(&self) -> usize` - возвращает размер пула
- `pub fn get(&self, index: u32) -> Rc<dyn ElementShader>` - возвращает шейдер по индексу, или fallback, если по индексу невозможно получить шейдер

#### Реализации трейтов

- Сгенерированная компилятором реализация `Clone`

### ShaderInput

```rust
#[derive(Clone, Debug)]
pub struct ShaderInput<'a> {
	pub uv: (f32, f32),
	pub normal: [f32; 3],
	pub luminance: f32,
	pub background_element: &'a Element,
	pub fragment_depth: f32,
	pub fragment_layer: u32,
	pub object_id: u32,
}
```

Структура, которая передаётся на вход шейдера, реализующего трейт `ElementShader`. Представляет собой бесцветный атомарный фрагмент дискретизации, который ещё не был записан `ElementBuffer`

| Поля                 | Описание                                                                                                             |
| -------------------- | -------------------------------------------------------------------------------------------------------------------- |
| `u`, `v`             | Текстурные координаты фрагмента, интерполированные из вершин треугольника или линии                                  |
| `normal`             | Интерполированная из вершин не нормализованная нормаль фрагмента                                                     |
| `luminance`          | Интерполированная из вершин освещённость фрагмента                                                                   |
| `background_element` | Неизменяемое значение элемента, который уже находится в `ElementBuffer` в позиции, куда может быть записан фрагмент. |
| `fragment_depth`     | Глубина фрагмента. Чем меньше это значение, тем фрагмент ближе к наблюдателю.                                        |
| `fragment_layer`     | Слой фрагмента, определяемый порядком плоскости в сцене. Чем позже плоскость, тем выше слой                          |
| `object_id`          | Идентификатор объекта, из которого берётся фрагмент                                                                  |

#### Реализации трейтов

- Сгенерированные компилятором реализации `Clone`, `Debug`

### ShaderOutput

```rust
#[derive(Clone, Debug)]

pub struct ShaderOutput {
	pub color: [u8; 4],
	pub luminance: Option<f32>,
	pub object_id: Option<u32>,
}
```

Структура, которая возвращается из шейдера, реализующего трейт `ElementShader`. Полученные оттуда значения будут записаны в `ElementBuffer`, если фрагмент находится ближе к наблюдателю, чем значение которое уже хранится в `ElementBuffer`.

| Поля        | Описание                                                                                                |
| ----------- | ------------------------------------------------------------------------------------------------------- |
| `color`     | Обязательное поле, определяющее цвет элемента в `ElementBuffer`                                         |
| `luminance` | Необязательное поле, которое может изменить яркость элемента в `ElementBuffer`                          |
| `object_id` | Необязательное поле, которое может изменить записанный идентификатор объекта в элементе `ElementBuffer` |

#### Реализации трейтов

- Сгенерированные компилятором реализации `Clone`, `Debug`

### Viewport

```rust
#[derive(Clone, Debug)]
pub struct Viewport {
pub x: f32,
pub y: f32,
pub width: f32,
pub height: f32,
pub scaling_mode: ScalingMode,
pub horizontal_alignment: HorizontalAlignment,
pub vertical_alignment: VerticalAlignment,
pub element_aspect_ratio: f32,
pub shader_map: Vec<u32>,
pub rotation_angle: f32,
pub buffer_offset_x: Option<u32>,
pub buffer_offset_y: Option<u32>,
pub buffer_width: Option<u32>,
pub buffer_height: Option<u32>,
}
```

Логический вьюпорт в координатах Plane, позиция которого определяется точкой в координатах (`x`, `y`), которая является верхним левым углом вьюпорта, а ширина и высота определяется в `width` и `height` соответсвенно.

| Поля                                 | Описание                                                                                                                                                                                      |
| ------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `x`, `y`                             | Координаты верхнего левого угла вьюпорта в логическом пространстве плоскости                                                                                                                  |
| `width`, `height`                    | Соответсвенно ширина и высота вьюпорта                                                                                                                                                        |
| `scaling_mode`                       | Режим масштабирования изображения из этого вьюпорта в выходной буфер                                                                                                                          |
| `horizontal_alignment`               | Выравнивание по горизонтали                                                                                                                                                                   |
| `vertical_alignment`                 | Выравнивание по вертикали                                                                                                                                                                     |
| `element_aspect_ratio`               | Соотношение сторон одной дискреты в итоговом буфере (ширина / высота)                                                                                                                         |
| `shader_map`                         | Таблица перенаправления индексов шейдеров. Индекс в этой таблице — это `local_shader_id` из треугольника, а значение по этому индексу — глобальный индекс шейдера в `ShaderPool` сцены.       |
| `rotation_angle`                     | Определяет поворот в радианах по часовой стрелке относительно исходного положения. В исходном положении вектор (x + width, y) сонаправлен оси X+, а вектор (x, y + height) сонаправлен оси Y+ |
| `buffer_offset_x`, `buffer_offset_y` | Смещение вьюпорта в координатах буфера                                                                                                                                                        |
| `buffer_width`, `buffer_height`      | Размер вьюпорта в координатах буфера                                                                                                                                                          |
#### Методы

- `pub fn new(x: f32, y:f32, width: f32, height: f32) -> Self` - возвращает новый `Viewport`, с заданными размерами и расположением
- `pub fn new_with_scaling(x: f32, y: f32, width: f32, height: f32, scaling_mode: ScalingMode) -> Self` - возвращает новый `Viewport`, с заданными размерами, режимом масштабирования и расположением
- `pub fn new_with_aspect(x: f32, y: f32, width: f32, height: f32, element_aspect_ratio: f32) -> Self` - возвращает новый `Viewport`, с заданными размерами, соотношением сторон одной дискреты и расположением
- `pub fn with_buffer_offset(mut self, x: u32, y: u32) -> Self` - Изменяет и возвращает экземпляр вьюпорта с заданным смещением выходного буфера
- `pub fn with_buffer_size(mut self, width: u32, height: u32) -> Self` - Изменяет и возвращает экземпляр вьюпорта с заданными размерами выходного буфера

### ScalingMode

```rust
#[derive(Copy, Clone, Debug, PartialEq)]
pub enum ScalingMode {
	Stretch,
	Contain,
	Cover,
	None,
}
```

Используется для определения масштабирования при несоответствии размеров `viewport` и выходного буфера

| Значение  | Описание                                                         |
| --------- | ---------------------------------------------------------------- |
| `Stretch` | Растянуть до размеров выходного буфера, игнорируя пропорции      |
| `Contain` | Вписать с сохранением пропорций, пустые места заливаются фоном   |
| `Cover`   | Заполнить всё с сохранением пропорций, обрезая выступающие части |
| `None`    | Не масштабировать, координаты 1:1                                |

#### Реализации трейтов 

- Сгенерированные компилятором реализации `Copy`, `Clone`, `Debug`, `PartialEq`
